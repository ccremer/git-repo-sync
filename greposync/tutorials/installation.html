Installation

Several installation methods are supported. Choose one that fits best your workflow.

All snippets are made for linux with amd64 architecture. Please adapt the links and commands to your OS and platform.
Tools used

  • wget (for downloading artifacts)

  • grep (to extract information from files)

  • gpg (for verifying downloaded artifacts)

  • docker (for Docker installation method)

  • sudo (to gain elevated privileges for installing)

Run in Docker

  1. Create alias

    gsync() {
  docker run \
    --rm \
    --user="$(id -u)" \
    --env GITHUB_TOKEN \
    --env HOME=/app \
    --env SSH_AUTH_SOCK=/tmp/ssh_agent.sock \
    --volume "${SSH_AUTH_SOCK}:/tmp/ssh_agent.sock" \
    --volume "${HOME}/.ssh/config:/app/.ssh/config:ro" \
    --volume "${HOME}/.ssh/known_hosts:/app/.ssh/known_hosts:ro" \
    --volume "${HOME}/.gitconfig:/app/.gitconfig:ro" \
    --volume "${PWD}:/app/data" \
    --workdir /app/data \
    ghcr.io/ccremer/greposync:${GREPOSYNC_VERSION:=latest} \
    $*
}

    • If any of the ~/.ssh files or ~/.gitconfig don’t exist, they are created as directories! This won’t work, so please ensure that these files exist before.

    • git clone inside the container fails if using git+ssh URLs when the SSH config files have not the correct permissions. Ensure that they are at least 0644 or lower.
