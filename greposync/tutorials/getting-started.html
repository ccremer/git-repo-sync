Getting started

üèÅ Goals

  1. Set up a template repository

  2. Onboard a managed repository

  3. Render the templates

  4. Influence templates

Prerequisites

You need the following tools installed in your $PATH.

  • git

  • gsync (See Installation)

  • editor (Your favorite text editor)

Additionally, you need the following.

  • A user account on github.com

  • SSH key associated with your GitHub account. See this article if you need help.

For the purpose of this guide, following repository names are being used:

  • greposync-control: The repository containing the config and templates.

  • my-repository: The test repository which we are going to be managing with greposync

Setup template repository

In the first step, we’re going to initialize a Git repository that contains the template.

  1. Create a new repository called greposync-control in GitHub

    Select Add a README file option, as we need an initial commit in a branch.

  2. Clone the new repository

    git clone <git-url>

  3. Initialize greposync

    cd greposync-control
gsync init

    You should now see a structure similar to the following:

    editor greposync.yml

Onboard a managed repository

  1. Create a new repository called my-repository in GitHub

  2. Configure managed_repos.yml (Optional)

    editor managed_repos.yml

    This step is required if you didn’t name your repository my-repository. my-repository is the default in managed_repos.yml when doing gsync init.

Render the templates

  1. Run greposync

    gsync update --dry-run=commit

    There is a warning about a missing .sync.yml. We will add this file in a later step.

Influence templates

  1. Create the .sync.yml file and override a value.

    pushd repos/github.com/*/my-repository &> /dev/null
cat <<EOF > .sync.yml
README.md:
  description: This is my new description!
EOF
git add -A
git commit -m "Add .sync.yml"
popd &> /dev/null

    We didn’t just randomly add a description field. The template in template/README.md.tpl actually has {{ .Values.description }} at the end. This tells the template engine to place our new value there, which in turn is coming from config_defaults.yml as a default value. Since we want to override the description for this repository, we provide it through .sync.yml that now lives in my-repository itself.

  2. Run greposync again

    gsync update --dry-run=commit

Inspect the result
