Add comment headers to files

‚ùì Question

For every templated file I want to add a comment header. Not all files have the same commenting syntax. How can I add such comments?

üìù Use case

Files should contain comments indicating that users shouldn’t modify the file at hand. If greposync runs again, it would overwrite their changes.

üí° Solution

Add the _helpers.tpl file to the template directory. It could contain the following content:

template/_helpers.tpl
{{- define "comment" -}}
{{- if .Values.comment.text }}
{{- with .Values.comment.open -}}
{{ . }}
{{ end -}}
{{- with .Values.comment.text -}}
{{ $.Values.comment.prefix }}{{ . | splitList "\n" | join (cat "\n" $.Values.comment.prefix | replace "\n " "\n") }}
{{ end }}
{{- with .Values.comment.closed -}}
{{ . }}
{{ end -}}
{{- end -}}
{{- end -}}

Configure the globals so that it allows flexible prefixes or enclosed comments.

config_defaults.yml
:globals:
  comment:
    text: | (1)
      This file is managed by greposync.
      Do not modify manually.
      Adjust variables in `.sync.yml`.
    prefix: ""
    open: <!--
    closed: -->
1 Simply setting text="" will disable the comment.
template/README.md.tpl
{{- template "comment" . -}}
# managed-repository

This results in a file rendered like this:

README.md
<!--
This file is managed by greposync.
Do not modify manually.
Adjust variables in `.sync.yml`.

-->
# managed-repository
