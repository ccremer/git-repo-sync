Create a pull request

üèÅ Goals

  1. Push an update to GitHub

  2. Create a new pull request

This tutorial continues where Getting started left. Be sure to complete that one first!

Prerequisites

You need the following tools installed in your $PATH.

  • git

  • gsync

  • editor (Your favorite text editor)

Additionally, you need the following.

  • A user account on github.com

  • SSH key associated with your GitHub account. See this article if you need help.

  • Personal Access Token (PAT) to authenticate with GitHub API. Following scopes are required:

    • repo

    • workflow

Export the PAT
export GITHUB_TOKEN=<your-github-token>

For the purpose of this guide, following repository names are being used:

  • greposync-control: The repository containing the config and templates.

  • my-repository: The test repository which we are going to be managing with greposync

Push an update to GitHub

  1. Run greposync again, but this time also push to remote

    gsync update --dry-run=push

  2. Inspect the Git log

    pushd repos/github.com/*/my-repository &> /dev/null
GIT_PAGER=cat git log --oneline
popd &> /dev/null
    Example Output
    3637433 (HEAD -> greposync, origin/greposync) Update from greposync (1)
e6f6b79 Add .sync.yml
4bb2b9f Update from greposync
16ef4fe (origin/main, origin/HEAD, main) Initial commit
    1 As you now can see, we have pushed the branch to origin.

Create a new pull request

  1. Edit greposync.yml and set pr.create=true

    editor greposync.yml

  2. Run greposync again, but this time no more dry run

    gsync update

Inspect the result

The update command should have printed the URL of your new pull request. Visit the pull request on github.com and verify that it correctly applied the following settings:

  • Subject

  • Description

  • Labels

If the commit branch and target branch stay unchanged, greposync should find and update the same pull request when you re-run the update with modifications.

Tutorial complete!
