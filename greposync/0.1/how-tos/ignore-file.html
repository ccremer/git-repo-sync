Ignore file in a repository

‚ùì Question

In a repository I maintain a special version of a file that cannot be easily templated. How can I tell greposync to ignore this file?

üìù Use case

Normally I have Makefile in the template but in example-repository that file is too deviating to be managed by the template.

üí° Solution

Add the unmanaged: true special value to a file. greposync will not touch that file when updating.

.sync.yml in example-repository
Makefile:
  unmanaged: true

You can ignore whole directories when appending / to the path:

my-dir/:
  unmanaged: true
