Main Configuration

greposync.yml holds configuration variables that define how greposync behaves when invoked.

All supported settings with defaults
git:
  commitBranch: greposync-update
  commitMessage: Update from greposync
  defaultNamespace: github.com
  forcePush: false
  root: repos
log:
  showDiff: false
  showLog: false
pr:
  body: This Pull request updates this repository with changes from a greposync template
    repository.
  create: false
  labels: []
  subject: Update from greposync
  targetBranch: ""
template:
  root: template
git.commitBranch

The branch name to create and checkout when making commits.

The simplest workflow with master branch looks like this:

  1. Clone repository with master branch

  2. Create and checkout greposync branch

  3. Update repository and make commits in greposync branch

  4. Push greposync branch

  5. Create pull request that merges greposync back into master
pr.targetBranch

The branch name which pull requests should be merged into. If empty, it defaults to git.defaultBranch (usually master or main).

pr.bodyTemplate

The pull request description text. Supports Go templates and the .Metadata.Repository variables

pr.labels

This parameter takes a string array of labels to apply on pull requests. Label names that don’t exist are created with an empty description and a random color. Foreign labels in existing pull requests are not removed or renamed.

Sync Labels In All Repositories

greposync can synchronize issue and pull request labels in all managed repositories. Configure the repositoryLabels key with config like this:

repositoryLabels:
  greposync:
    color: '#ededed'
    delete: false
    description: updates from template repository
    name: greposync

The keys within repositoryLabels.* have no real value and are for description only. This structure was favored over an array in order to make this configuration deep-mergeable in the future.
